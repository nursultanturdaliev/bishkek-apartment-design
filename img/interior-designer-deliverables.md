## 2. Фазы проектирования и минимальный состав результатов

| Фаза | Минимальный состав результатов |
|------|-------------------------------|
| **Концепция / mood** | Нарратив, принципы, референс-доска, предварительная палитра, ориентировочный бюджетный коридор. |
| **Рабочее проектирование (DD)** | Отработанные планы, ключевые разрезы/фасады, ориентировочные спецификации, матрица помещений, уточнённый бюджет. |
| **Исполнительная документация (CD)** | Чертежи и ведомости, пригодные для сметы/строительства в рамках интерьера; узлы примыканий — по согласованию. |
| **Стройка и монтаж** | Запросы информации (RFI), журнал рассмотрения рабочих чертежей заводов, указания на объекте, список замечаний, punch list. |
| **Передача** | План «как построено» / исполнительная схема, ведомости, реестр гарантий, эксплуатация и уход, запас материалов, фотоотчёт. |

<a id="section-2-1"></a>

### 2.1 Типовой объём дизайна интерьера (фиксировать в договоре)

Ориентир по строкам работ (включить / исключить явно):

- планировочные решения, зонирование, расстановка мебели (фиксированной и свободной);
- спецификация архитектурных отделок (полы, стены, потолки) в границах интерьера;
- проектирование мебели (кухня, шкафы, хранение в холле) с рассмотрением рабочих чертежей производства;
- спецификация FF&E (или сопровождение закупки);
- концепция освещения и ведомость светильников (увязка с потолком и мебелью);
- требования к солнцезащите / шторам / жалюзи — монтажные узлы с поставщиком;
- выезды на объект, мокапы, согласование образцов, сопровождение снабжения, актуальные замечания, punch list.

**Допущения, исключения, зона подрядчика/заказчика** — в [требованиях к проекту, §3](./apartment-project-requirements.md#section-3).

## 3. Категории работ (чеклист для договора / DD–CD)

Каждую применимую строку закрыть в DD/CD или явно пометить «Н/П» с обоснованием.

| Область | Требование |
|---------|------------|
| **Бриф** | Согласованный бриф заказчика (см. [§5.1 требований](./apartment-project-requirements.md#section-5)): образ жизни, хранение, бюджет/сроки, ограничения, «must keep». |
| **Планировка** | Зонирование, маршруты, эргономика, гибкость, входная группа, приватность (шум из холла). |
| **Отделка** | Полы, стены, потолки; **концепция** гидроизоляции мокрых зон (узлы — по норме и с сертификацией при необходимости). |
| **Мебель / корпус** | Кухня, шкафы, хранение в холле, кромки, встройка техники. |
| **FF&E** | Мебель, посадка, кровати, обеденная группа; пропорции по обмерным размерам. |
| **Освещение** | Качественные цели и сценарии — в [требованиях, §5.6](./apartment-project-requirements.md#section-5-6); **ведомость** и совместимость диммер–драйвер — по [§4.3](#section-4-3) ниже; увязка с потолком и мебелью. |
| **ОВиК (зона дизайнера)** | Розетки / данные / ТВ; логика выключателей; привязка кондиционеров — **с проектировщиком ОВиК**; **газовый котёл и радиаторы** — привязка к мебели и отделке **с газовым монтажом** (не переносить узел без него). |
| **Окна и шторы** | Приватность, солнцезащита, сон, акустика (состав стека / подкладка по необходимости). |
| **Цвет и материалы** | Логика палитры; зоны износа; регламент уборки. |
| **Акустика и комфорт** | Ковры, текстиль, уплотнители дверей по необходимости. |
| **Декор и искусство** | После функции; акценты и масштаб. |
| **Устойчивость и здоровье** | Вентиляция; низкая эмиссия ЛКМ где уместно; долговечность вместо одноразовых решений. |
| **Закупка и реализация** | Запросы котировок, рабочие чертежи заводов, мокапы, выезды, punch list. |

## 4. Спецификации — минимальный состав полей ведомостей

Ведомость нельзя считать «полной» без перечисленных столбцов, если поле не **Н/П** (указать причину).

<a id="section-4-1"></a>

### 4.1 Ведомость отделки

Помещение / элемент • материал • производитель / линейка • цвет / наименование • рисунок / раппорт • габариты • затирка / герметик • класс противоскольжения (мокрые зоны) • поставщик • согласование образца (да/нет)

<a id="section-4-2"></a>

### 4.2 Ведомость FF&E

ID позиции • ссылка на изображение • кол-во • габариты • отделка / ткань • артикул (если известен) • поставщик • срок поставки • монтаж (подрядчик / заказчик) • гарантия • уход

<a id="section-4-3"></a>

### 4.3 Ведомость освещения

**Минимум (обязательные столбцы):**  
ID светильника • тип • место / привязка к плану • патрон / модуль • целевые CCT/CRI • диммирование / управление • примечание по расположению драйвера • ограничение по высоте потолка

**Рекомендуемые дополнительные поля (уровень «лучшая практика» / ориентир 2026):**  
поток **лм** • потребляемая мощность **Вт** • **Ra / R9** (и при наличии **TM-30 Rf/Rg**) • тип линзы / угол • **UGR** (если применимо) • класс **IP/IK** • тип драйвера (в т. ч. flicker / **Pst LM**) • совместимость с диммером (марка/модель или тип: Trailing Edge / 0–10 V / DALI) • ссылка на **IES/LDT** • цвет корпуса / reflector • срок поставки • запасной **LED-модуль** (Y/N)

## 5. Состав пакета передачи заказчику (handover)

Пакет согласовать с оглавлением PDF к закрытию объекта. Типовой состав:

| № | Документ / пакет | Краткое содержание |
|---|------------------|-------------------|
| 1 | Концепция / концепт-бук | История, принципы, визуал (если не выдано ранее). |
| 2 | **Планы и разрезы «как построено»** | Планы, фасады, ключевые узлы = **фактическое** состояние. |
| 3 | Ведомость отделок | По комнатам или по элементам; ссылки на спеки и образцы. |
| 4 | Ведомость FF&E | См. [§4.2](#section-4-2). |
| 5 | Ведомость освещения | См. [§4.3](#section-4-3). |
| 6 | Спецификации / листы данных | Сшиватель PDF или структурированное приложение. |
| 7 | Свод по мебели | Перечень утверждённых рабочих чертежей завода + номера ревизий. |
| 8 | Реестр гарантий | Изделие, серийный номер при наличии, срок, контакт, порядок претензии. |
| 9 | Руководства по эксплуатации и уходу | Техника, камень, дерево, покрытия. |
| 10 | Запас / «чердачный» склад | Ручки, плитка, коды краски, обои. |
| 11 | Фотоотчёт | Готовые помещения — для страховки и будущих ремонтов. |
| 12 | Punch list | Подписанное закрытие; ожидания по дефектам / гарантии (**формулировки — проверить с юристом по местному праву**). |
| 13 | Умный дом / АВ | Краткое руководство и сброс, если применимо. |

## 6. Координация смежных лицензируемых специалистов

На уровне **координации** остановиться и назначить ответственную дисциплину для:

- несущих стен, плит, проёмов, усиления;
- **сертификации** гидроизоляции мокрых зон («тэнкинг»);
- газового оборудования и трасс газа (в т. ч. **автономного газового отопления** — котёл, ввод, дымоудаление);
- модернизации главного щита, трёхфазного ввода;
- изменений фасада / оболочки, влияющих на конденсацию или пожар;
- противопожарного разделения между квартирами / путей эвакуации.

**Формулировка для спецификаций:** *Согласовать с лицензированным архитектором / инженером / проектировщиком ОВиК в юрисдикции объекта; работы по **газу** — с **газовой службой / организацией с допуском**; при необходимости уточнить у уполномоченного органа.*

## 7. Ограничение ответственности (дисклеймер)

Документ — **вспомогательный шаблон** для структурирования договорных ожиданий и пакета передачи по интерьеру. Это **не** юридическая консультация, не расчёт несущих конструкций и не сертификация соответствия нормам. Конструктивную часть, пожарную и газовую безопасность, проектирование **систем** электроснабжения должны проверить **квалифицированные специалисты с допуском** в юрисдикции объекта. Заказчик и подрядчики обязаны **проверить размеры на объекте**, совместимость с инженерией здания и получить **официальные согласования**, где требуется. Формулировки по гарантиям и дефектам должны соответствовать **местным договорам и закону** — при необходимости привлечь юриста.
